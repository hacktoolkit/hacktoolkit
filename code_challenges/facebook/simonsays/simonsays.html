Do you like puzzles? So do we.

If you love puzzles like we do, become a fan of the new Puzzle Master Facebook Page. Notes are regularly posted to answer questions, explain puzzles, and announce new things. While you're here, try your hand at the following puzzles. The larger the difficulty, the harder it gets (hors d'oeuvres are simple tests to help you out).

Simon Says

Simon Says is intended to be a relatively simple introduction to use the Thrift remote-procedure call framework to easily write server/client software. This eponymous puzzle follows the rules for Simon (the electronic toy) by Milton Bradley (now Hasbro).


Game Rules

Your client begins the game by connecting to a Simon Says Thrift server. This is done for you if you email your submission to the robot for evaluation. The client must call the startTurn API to receive a list of colors from the server. The client must then play back these colors in the correct order to the server using the chooseColor API. Once done, the client must then call endTurn.

If endTurn returns false, the game has not yet been won. The client should repeat the process of calling startTurn, chooseColor several times, and then endTurn. Once endTurn returns true, the client is in a win-ready state, and may win the game by calling the winGame API. Once invoked, the client may then safely terminate the connection and shut down.

If the client chooses the wrong color, calls an extra color when it should have called endTurn, or calls endTurn prematurely, the server will restart its state. The server resets the color sequence back to the starting length of one and will return this new color sequence at the next startTurn call. If your client detects this error, it should stop and immediately start over by calling startTurn.


Special Submission Directions

The service specification (simonsays.thrift) can be compiled via the Thrift compiler into client and server skeleton code, classes, and/or stubs depending on the target language of choice. Your submission will differ from traditional puzzles in that a bash script is provided for you to submit with your code. If you do not use this provided bash script, your submission will not be graded properly.

You are also responsible for providing a binary file named simonsays_client which must either already exist, or be buildable using the commands make or ant similar to a normal submission. Just like regular submissions, this binary file can merely be a script to execute your actual client program. Any Thrift resources required by your client must also be included in your submission or built by your submission. Your client may be written in any accepted language that is also Thrift compatible. The Thrift libraries and includes have been installed in the default locations on the machine.

The simonsays_client must take exactly two arguments; the first argument is the hostname of the Simon Says server in string form and the second argument is the port number to connect to.

Example Simon Says client startup: 
./simonsays_client thriftpuzzle.facebook.com 9030


Resources

You can get the version of Thrift used by the Puzzle Master by visiting the Facebook Developers web site and downloading the current archived release.

The Simon Says Thrift service file can be downloaded from Facebook's site.

The Public Thrift Test Server is available for your use in developing your client. Server logs are also available in addition to whatever debugging you put in your client.

Submission directions

All submissions must execute in a *NIX type environment (sorry, no Windows specific solutions are accepted). You are not guaranteed any libraries or plugins beyond what is part of the language/interpreter itself. The following languages are accepted:
  • GNU C/C++ 4.2.3
  • Ericsson Erlang 5.5.5
  • GHC Haskell 6.8.2
  • Sun Java 1.5.0_15
  • INRIA OCaml 3.10.0
  • Perl 5.8.8
  • PHP 5.2.4
  • Python 2.5.2
  • Ruby 1.8.6
Some puzzles may require being solved with the following libraries:
  • Thrift r760184
